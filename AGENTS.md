# Agent Operating Guide

You are working in an async code research repository.

## Mission

Produce high-signal, auditable research artifacts that help a human make implementation decisions quickly.

## Workflow

1. Start from a clear question (usually a GitHub issue).
2. Break work into verifiable subtasks.
3. Record findings in `research-notes/YYYY-MM-DD-topic-slug.md`.
4. Link primary sources for all major claims.
5. Distinguish facts, inferences, and open questions.
6. Commit artifacts in logical increments.

## Output quality bar

- Be precise and concrete.
- Include trade-offs and risks.
- Call out unknowns immediately.
- Do not invent technical details.
- Prefer simple recommendations.

## Standard note template

Use this structure for each research note:

- Question
- Context
- Findings
- Options
- Recommendation
- Risks
- Open questions
- Sources
